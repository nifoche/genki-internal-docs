# ドキュメントレビューチェックリスト

ドキュメントを公開する前に、以下のチェックリストを確認してください。

## セキュリティチェック

### 機密情報の確認

- [ ] **APIキー**: `sk-`, `API_KEY`, `api_key`, `AKIA...` (AWS) 等の文字列が含まれていない
- [ ] **パスワード**: `password`, `PASSWORD`, `passwd`, `pass` 等が含まれていない
- [ ] **トークン**: `token`, `TOKEN`, `secret`, `SECRET` 等が含まれていない
- [ ] **個人情報**: 電話番号、メールアドレス、住所等が含まれていない
- [ ] **データベース接続情報**: データベースURL、ユーザー名、パスワードが含まれていない
- [ ] **SSH鍵**: `-----BEGIN`, `PRIVATE KEY`, `ssh-rsa` 等が含まれていない
- [ ] **Cookie/セッション**: `Cookie:`, `session-id` 等が含まれていない

### コード内のシークレット

- [ ] ハードコードされた認証情報がない
- [ ] 環境変数の例が適切にサニタイズされている（`your-api-key-here` 等）
- [ ] `.env` ファイルの内容が含まれていない

## コンテンツチェック

### リンクの確認

- [ ] 内部リンクがすべて正しく動作する
- [ ] 外部リンクがアクセス可能である
- [ ] 画像へのリンクが正しい

### 書式の確認

- [ ] Markdownの書式が正しい
- [ ] 見出し（`#`）の階層が適切である
- [ ] コードブロックが適切にフォーマットされている

### コンテンツの品質

- [ ] 誤字脱字がない
- [ ] 専門用語に適切な説明がある
- [ ] 文書が最新の状態である

## 自動チェックツール

以下のコマンドでシークレットの自動チェックを実行できます：

```bash
# シークレット検出（簡易版）
grep -r "sk-" docs/
grep -ri "password" docs/
grep -ri "api_key" docs/
grep -ri "secret" docs/

# より高度なチェック（必要に応じてインストール）
# npm install -g git-secrets
# git-secrets --scan
```

## レビュープロセス

1. **自己レビュー**: このチェックリストをすべて確認
2. **ピアレビュー**: チームメンバーによるレビューを受ける
3. **テスト**: ローカルでビルドして表示を確認
4. **コミット**: 以上の工程を経てからコミット

## 例外の取り扱い

機密情報を含める必要がある場合は、以下の対策を講じてください：

- **環境変数化**: `.env.example` にのみテンプレートを記載
- **外部ドキュメント**: 機密情報は別のプライベートリポジトリで管理
- **プレースホルダー**: `your-api-key-here` 等の明確なプレースホルダーを使用

## 緊急時の対応

誤って機密情報をコミットしてしまった場合：

1. **即座に修正**: 誤った情報を削除・置換
2. **コミット修正**: `git commit --amend` で最新コミットを修正
3. **強制プッシュ**: `git push --force` で上書き（既に公開されている場合は要注意）
4. **シークレット無効化**: APIキー等は直ちに無効化
5. **関係者への通知**: 必要に応じてチームメンバーに通知

---

**重要**: このチェックリストは、すべてのドキュメントのプルリクエスト時に適用されます。
